================================================================================
  WORKER PROCESSING PIPELINE
================================================================================

Image Upload
    ↓
Publish to 4 Queues
    ├─ resize_queue (5 sec processing)
    ├─ thumbnail_queue (3 sec processing)
    ├─ ocr_queue (7 sec processing)
    └─ ai_tagging_queue (6 sec processing)
    ↓
Parallel Processing
    ├─ Fair dispatch with prefetch_count=1
    ├─ Automatic load balancing
    └─ Max total: ~7 seconds (parallel)
    ↓
Result Handling
    ├─ Success → ACK (remove from queue)
    ├─ Failure + retries left → Requeue
    └─ Failure + max retries → Send to DLQ

================================================================================
  TROUBLESHOOTING
================================================================================

RabbitMQ Connection Failed?
→ docker-compose ps rabbitmq
→ docker-compose restart rabbitmq
→ Check logs: docker-compose logs rabbitmq

API Not Responding?
→ docker-compose ps api
→ docker-compose logs api
→ Restart: docker-compose restart api

Workers Not Processing?
→ docker-compose ps (verify all workers running)
→ docker-compose logs resize_worker_1
→ Check RabbitMQ UI: http://localhost:15672

Port Already in Use?
→ Find: lsof -i :8000
→ Kill: kill -9 <PID>

See DEVELOPMENT.md for detailed troubleshooting guide
